SQUAD.md

The open standard for multi-agent team orchestration

Read the Spec Get the CLI

AGENTS.md tells agents how to work on YOUR project.

But who tells agents how to work WITH EACH OTHER?

The Problem

Single agents hit limits. Enterprises need teams, not individuals. But there's no standard for multi-agent coordination.

SQUAD.md defines multi-agent teams: roles, coordination patterns, triggers, and budgets.

How It Works

SQUAD.md uses YAML frontmatter with a Markdown body. Simple enough to write by hand, expressive enough for complex workflows.

---
schema: squad/v0.1
name: engineering
mission: Ship reliable software.
budget:
  daily: 50
triggers:
  - name: pr-review
    agent: code-reviewer
    condition: "event: pull_request.opened"
---

# Engineering Squad

## Agents
| Agent | Role | Trigger |
|-------|------|---------|
| code-reviewer | Review PRs | event |
| ci-optimizer | Fix builds | event |

## Workflow
1. PR opened triggers code-reviewer
2. CI failure triggers ci-optimizer
3. Weekly debt report on Mondays

Key Features

Works with any AI

Vendor-neutral specification. Claude, GPT, Gemini, local models.

Just Markdown

YAML frontmatter + markdown body. No special tools required.

Battle-tested

Agents Squads runs their entire company with this format.

Budget Controls

Set daily, weekly, monthly spend limits per squad.

"We don't want users paying for infra that might not produce expected outputs." — Jorge Vidaurre, Agents Squads

Get Started

# Install the CLI
npm install -g squads-cli

# Initialize in your project
squads init

# Check your squads
squads status
